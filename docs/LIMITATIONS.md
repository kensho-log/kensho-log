# 本検証の限界（固定テンプレ）

各検証動画の末尾および各 Issue / PR 本文に、以下の項目をチェックリスト形式で必ず記載する。  
未評価の項目は「未評価」と明記し、空欄にしないこと。

---

## データ範囲の限界

- [ ] 対象期間（開始日・終了日）
- [ ] 対象市場 / 通貨
- [ ] 対象銘柄・指数の選定理由
- [ ] 生存者バイアスの扱い（上場廃止銘柄の包含 / 除外）
- [ ] 通貨換算の有無と為替レートの取得元・タイミング

## データソースの限界

- [ ] 一次ソース名（例: yfinance / Stooq / JPX / FRED）
- [ ] 取得日時（UTC）
- [ ] データソース間の差分 %（> 0.1% の場合は警告停止）
- [ ] 欠損値・分割・配当調整の扱い

## 計算モデルの限界

- [ ] 税制の反映有無（NISA / 特定口座 / 海外課税）
- [ ] 信託報酬・売買手数料・スプレッドの反映有無
- [ ] リバランス頻度と閾値
- [ ] 配当再投資の扱い（即時再投資 / 税引後再投資）
- [ ] 現実の約定遅延・スリッページの扱い

## 前提・除外

- [ ] シミュレーションに含めなかった要因（インフレ調整・相続・贈与税・退職金制度等）
- [ ] 外生ショック（戦争・パンデミック等）の特別扱いの有無

## 再現性

- [ ] 本検証コードの Git コミットハッシュ
- [ ] 使用した `requirements.txt` の内容（バージョン固定）
- [ ] 実行環境（Python バージョン / OS）

---

本リスト未記載の要因は、結果に影響し得る未評価の変数として扱う。  
視聴者は自身のユースケースに対する外挿可能性を、上記の限界を踏まえて自己判断すること。
